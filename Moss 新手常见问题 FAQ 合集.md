# Moss 新手常见问题 FAQ 合集｜环境、开发、贡献全答疑
## 一、基础认知类
### Q1：Moss 到底是什么，核心作用是什么？
A：Moss 是 Monad 生态开源 AI Agent × Web3 中间层，作为 AI 智能体与区块链之间的桥梁，封装各类链上协议、交易构造逻辑，大幅降低 AI 调用链上操作的开发成本，支持本地零资金模拟测试，面向开发者、运营者开放开源贡献。

### Q2：Moss 和普通链 SDK、智能合约工具的区别在哪？
A：普通 SDK 仅提供底层链交互接口，需要手动编写交易、适配每个协议；Moss 自动检索并封装链上所有 DeFi 协议，内置交易模拟校验、AI 代码开发规范，专门为 AI Agent 场景做优化，无需重复造轮子。

### Q3：零基础小白也能参与 Moss 开源贡献吗？
A：可以。项目分技术、运营两大贡献赛道，运营方向无需代码能力，仅需撰写文档、整理教程、维护社群；开发方向提供标准化模板，可借助大模型辅助编码，本地模拟环境无需真实加密资产。

## 二、环境安装与本地运行类
### Q1：本地运行 Moss 需要什么硬件/工具？
A：需要 Git、Monad 定制版 Foundry，终端环境即可；推荐搭配 MATT 工具规范 AI 开发流程，无需部署完整本地节点，支持 fork 测试网模拟交易。

### Q2：克隆仓库后执行脚本报错，大概率是什么原因？
A：1. Foundry 版本不匹配，需安装 Monad 适配版本；2. 依赖未完整拉取，执行依赖更新命令；3. 分支未切换至开发主分支，请勿在 main 主分支直接修改代码。

### Q3：本地测试是否需要充值真实代币？
A：完全不需要。项目 example 目录内置本地 fork 模拟环境，所有交易仅在本地模拟执行，不存在真实资金损耗，适合新手反复调试。

### Q4：本地 Demo 跑通后，如何确认流程完整可用？
A：完整走完四步流程：协议发现→交易构造→NCP 模拟校验→模拟钱包执行，无报错、校验结果与预期操作一致，即代表环境正常。

## 三、开发与AI编码相关
### Q1：可以直接用大模型生成代码提交 PR 吗？
A：不可以。AI 生成代码仅作初稿，开发者必须人工通读审计、本地全量测试，修复逻辑漏洞后，才可提交 PR，未审计代码会被审核驳回。

### Q2：新增协议适配器有固定模板吗？
A：有。仓库 package 目录内置通用 EIP、DeFi 协议适配器模板，同时配套 10 份 ADR 开发规范文档，直接复用模板即可快速完成协议接入。

### Q3：开发时如何控制代码规范，避免格式问题？
A：项目内置自动化校验工具，提交前执行校验脚本，自动检测代码格式、逻辑漏洞；同时推荐使用 MATT 工具管控 AI 代码输出流程。

## 四、开源贡献 & PR 提交类
### Q1：提交 PR 有什么硬性规范？
A：1. 单 PR 仅处理单一需求，不混合多类修改；2. 提交前本地跑通全部测试用例；3. PR 描述清晰写明修改目的、对应 Issue；4. 遵循仓库代码命名、注释规范。

### Q2：没有开发能力，能做哪些贡献？
A：运营向贡献：撰写科普文章、整理FAQ/教程、录制操作演示、社群日常维护、策划AMA活动、收集用户问题反馈。

### Q3：提交贡献后有什么激励？
A：长期稳定贡献的开发者、运营者，可获得专项奖金、线下活动差旅报销、Monad 生态项目优先合作等权益。

## 五、社区与沟通渠道
### Q1：遇到问题去哪里提问、获取帮助？
A：海外社区：Discord（全球开发者交流）、Telegram；国内社群：项目官方微信群，可联系管理员入群咨询。

### Q2：如何查看当前开放的可贡献任务？
A：前往 GitHub 仓库 Issue 板块，官方会标注低门槛新手任务、协议适配需求、文档优化需求，可直接认领对应任务。

## 六、应用场景相关
### Q1：基于 Moss 能搭建哪些实用应用？
A：自动化链上收益 Agent、黑客松 AI+Web3 Demo、社群交互活动工具、多协议快速适配插件、教学演示案例。

### Q2：Moss 仅支持 Monad 链吗？
A：项目原生适配 Monad，架构支持扩展多链协议适配器，后续会逐步兼容更多公链。
